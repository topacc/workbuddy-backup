> Last updated: 2026-07-04T03:40:28+08:00

## Memory Block

**工作背景**
用户是直属海关财务处副处长，负责19个行政/公共单位的预算数据分析，日常工作涵盖预决算管理、政府采购、基建管理及财务分析与报告撰写。同时负责单位培训考核，正计划搭建线上扫码答题系统。技术方面，用户正在开发名为CollectSync的Android应用（采用合规采集路径），并致力于APK构建工具链的搭建。

**个人背景**
用户偏好“总体→单位级→单一单位→演示材料”的渐进式分析路径，要求报告具备全面的可视化图表及结构化呈现。交付物严格偏好纯HTML/CSS/JS单文件（样式脚本内联、禁止外部资源），视觉设计倾向橙红渐变晨报风格，时间显示偏好“人话格式”。在技术方案中，要求必须包含架构、数据流、权限及验证计划。用户习惯给出详细结构化生成要求，并使用ardot-design-assistant设计助手技能。行文角色定位需符合处级领导干部工作场景，体现政策意识与管理视角，表达正式、稳健、规范并尽量去AI化。

**当前关注**
正在制作“政府采购与招标投标防范围标串标及违法违纪”培训PPT，面向下属单位，并计划使用IMA知识库搜索相关法律法规作为素材。持续关注AI领域资讯，习惯通过@skill:"AI HOT"获取AI日报。在财务与技术项目中，重点在于流程精细化及利用技术提升行政管理效率，强调输出内容的逻辑严谨性与依据充分性。

**近期动态**
- 推进政府采购培训项目：制作面向下属单位的合规培训PPT，重点防范围标串标及违法违纪行为，并准备利用IMA知识库搜集法律法规范畴的素材。
- 确立正式文档交付标准：明确Word报告格式（小标宋二号题目、仿宋三号正文等）及DOCX渲染QA流程，偏好可视化核验以确保版式合规，必要时进行逐页检查。
- 完成技术环境配置：验证了humanizer-zh Codex技能与NotebookLM MCP，确立了优先使用ASCII临时路径及工作区内工具链的开发习惯，项目路径已支持中文字符。
- 明确交互与执行规范：要求在长任务期间提供定期进度报告；遇到安装阻断时，若用户回复继续，则应推进下一安全路径而非停留讨论；Literal bootstrap sequence应直接完成设置并验证本地URL。
- 推进CollectSync项目：Android应用开发采用稳定合规采集路径，激进自动化需明确启用，并优先在工作区内搭建构建工具链。
- 强化输出质量控制：正式报告需进行DOCX渲染/版式核验，关注图表裁切与分页；分析材料需突出形势判断、问题剖析及改进措施，保持专业、客观、克制的语气。
